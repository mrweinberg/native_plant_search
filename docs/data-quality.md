# Data quality decisions

The single reference for how the catalog's **derived / sourced data** is produced
and the judgment calls behind it. When you add a field sourced from an external
dataset, or make a non-obvious call about provenance, thresholds, or known
inaccuracies, record it here. Curated horticultural fields (light, soil, bloom…)
are not covered — see the `add-plants` skill for those.

Guiding principle: **prefer authoritative sources** (USDA PLANTS, Kew/WCVP, EPA
CEC ecoregions) over raw-occurrence proxies, and be honest in the UI about
confidence.

---

## `nativeStates` — native range by US state

**Legacy method (still the base):** `enrich-usda.mjs` marks a state native if
USDA says the species is native to the L48 *region* **and** GBIF shows ≥50
occurrences in that state. USDA only certifies nativity at the region level, and
GBIF occurrences include cultivated/escaped plants, so this proxy is wrong in
both directions:

- **Over-claims** naturalized garden escapes (e.g. *Yucca filamentosa* recorded
  "native" to Ohio — it's a Southeast native that escapes cultivation).
- **Under-claims** genuinely-native states with sparse observation records.

**Hardening with WCVP (the authority).** Kew's World Checklist of Vascular Plants
gives curated native-vs-introduced status per region (≈ US state). POWO's own API
is Cloudflare-walled, but the same WCVP data is on **GBIF**
(`/v1/species/{key}/distributions`, `source` = "World Checklist…",
`establishmentMeans` = NATIVE/INTRODUCED/MANAGED). GBIF's matcher handles the
name→taxon reconciliation.

A full-catalog dry-run (`scripts/_nativity-dryrun.mjs`) showed under-claiming is
widespread and that **naive removals are dangerous**: WCVP narrows some taxa
(recent splits, cosmopolitan/circumboreal species it treats as introduced in
North America), which would strip genuine natives (yarrow, bracken & lady ferns,
harebell, foamflower, partridgeberry…). So the applied policy is deliberately
conservative:

1. **Cosmopolitan guard** — if WCVP reports **zero** native US states for a
   species (a taxonomic disagreement, not a real absence), **keep ours unchanged**.
2. Otherwise: **add** every WCVP-native US state we lack, and **remove only the
   states WCVP explicitly flags `INTRODUCED`** (high-confidence naturalization).
   States merely *absent* from WCVP are kept — absence is ambiguous.
3. **No WCVP record** (~13% of the catalog) — keep ours, unchanged.

**Known limitation:** because absent ≠ introduced, some over-claims survive
(Yucca keeps Ohio). The range map softens these (states with no county-level data
render as a lighter "state-level" fill, not solid county shading). A complete fix
needs county-level native-vs-adventive data — **BONAP** — whose data is locked in
map images / licensing; that's the next escalation if needed.

Generated by `enrich-nativity.mjs`. Changing `nativeStates` **cascades**: re-run
`enrich-county.mjs` then `enrich-biomes.mjs` (both gate on it).

## `county-index/` — native range by county

Per-state inverted index (`county FIPS → plant indices`) from **USDA PLANTS county
distribution** (`POST /api/PlantProfile/getDownloadDistributionDocumentation`),
gated by `nativeStates`. It is **presence-only** — USDA gives no native/introduced
flag at county level — so accuracy rides on `nativeStates`. Built by
`enrich-county.mjs`; the manifest is **append-only** so indices survive plant
additions. **Connecticut has no USDA county data** (state-level only; it abolished
counties) — its chunk is empty and the map falls back to a whole-state fill.

## `nativeBiomes` — CEC Level I ecoregion ("biome")

Derived by overlaying each plant's county distribution on the **EPA/CEC North
America Level I ecoregions** (the authoritative biome framework): county centroids
→ point-in-polygon → biome. `gen-county-biomes.mjs` builds the committed
`county-biome.json` crosswalk; `enrich-biomes.mjs` aggregates per plant, keeping a
biome only if it covers **≥5%** of the plant's classified counties (drops the
naturalized tail). 1,733/1,770 plants get 1–3 biomes; the rest are state-level-only
counties or Hawaii (outside CEC North America). Inherits any `nativeStates` error,
so it improves automatically as nativity is hardened.

## Range map presentation

The plant-detail map is a **national county choropleth** (Albers USA). Counties
with USDA county data are solid; native states with only state-level USDA data
(e.g. Maryland, Connecticut) render as a **distinct lighter "state-level" fill** —
honest about lower confidence rather than implying confident statewide nativity.
